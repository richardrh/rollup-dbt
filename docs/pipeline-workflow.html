Restored Polars Pipeline Workflow

This document describes the current code path in this repository. The restored pipeline is a schema-driven Polars LazyFrame DAG, not a SQL/dbt runtime.

1. CLI and supported invocation

CLI availability: there is no formal pipeline CLI entrypoint in pyproject.toml ([project.scripts] is absent), and polars/rollup/pipeline.py has no main(), argparse, Click, or Typer command. The supported invocation path is Python API usage.

Call the orchestrator from Python with the repository root as root. Typical calls:

from rollup.pipeline import build_pipeline, collect_loss_summary

models = build_pipeline(root=".", report_path="data/output/report.xlsx")
loss_summary = collect_loss_summary(root=".")

Call path from invocation into polars/rollup/pipeline.py:

Python caller
  ├─ rollup.pipeline.build_pipeline(root=..., report_path=...)
  │    ├─ load_pipeline_schema()
  │    ├─ build_sources()
  │    ├─ preflight_pipeline_boundary()
  │    ├─ build_staging()
  │    ├─ build_intermediate()
  │    ├─ build_marts()
  │    └─ build_reports()
  └─ rollup.pipeline.collect_loss_summary(root=...)
       └─ build_pipeline(...).marts.loss_summary.collect()

5. Polars lazy queries and data operations by stage

StageReadsKey Polars operationsProduces
Source scansRequired CSV seeds; RiskLink/Verisk YLT parquet globs; optional canonical EP summary CSV globs if present.pl.scan_csv with schema overrides and date parsing for CSV; pl.scan_parquet for parquet path/glob scans.Raw source LazyFrames in PipelineSources.datasets.
Selected analyses stagingselected_analyses source CSV.select(pl.col("vendor").cast(pl.String), pl.col("analysis_id").cast(pl.String)).stg_selected_analyses: run allow-list.
YLT normalization stagingraw_risklink_ylt and raw_verisk_ylt.RiskLink select: literal vendor risklink, aliases anlsid/yearid/eventid/loss, casts analysis/loss. Verisk select: literal vendor verisk, aliases Analysis/YearID/EventID/GroundUpLoss, casts analysis/loss. pl.concat(..., how="vertical").stg_normalized_ylt: canonical vendor, analysis, year, event, loss rows.
Select lossesNormalized YLT and staged selected analyses.join(..., on=["vendor", "analysis_id"], how="inner").int_selected_losses: only selected analysis/event losses.
Enrich lossesSelected losses plus analyses, perils, lobs.Dimension selects for analysis/peril/LOB columns, then left joins on [vendor, analysis_id], peril_id, and lob_id.int_enriched_losses: losses with peril, LOB, office/class/currency metadata.
Apply VOR adjustmentsEnriched losses plus forecast_factors, fx_rates, euws_rate_factors.Forecast select(class, office, factor); FX filter(target_currency == "GBP") then select(currency_code→currency, rate→fx_rate); EUWS select(model_event_id→event_id, occ_year→year_id, factor). Left joins, with_columns(coalesce(..., 1.0)), then with_columns(loss * forecast_factor * fx_rate * euws_rate_factor).int_adjusted_losses: enriched rows with defaulted factors and adjusted_loss.
Blend lossesAdjusted losses plus blending_weights.filter(return_period == 0), select(peril_id, vendor, base_model, weight→blend_weight), left join on [peril_id, vendor], default missing weight to 1.0, compute blended_loss = adjusted_loss * blend_weight.int_blended_losses.
Analysis loss martint_adjusted_losses.group_by(vendor, analysis_id, peril_id, peril_name, rollup_lob, currency); aggregate sum(loss), sum(adjusted_loss), len(); sort.mart_analysis_loss_summary; mart_loss_summary is a compatibility select(vendor, analysis_id, total_loss, event_count) from it.
Event fanout martint_adjusted_losses.select event, peril, rollup, currency, factor, and adjusted loss columns; sort(vendor, analysis_id, year_id, event_id).mart_event_loss_fanout.
Blended summary martint_blended_losses.group_by(peril_id, peril_name, vendor, base_model); aggregate max(blend_weight), sum(adjusted_loss), sum(blended_loss); sort(peril_id, vendor).mart_blended_loss_summary.
Reportsmart_loss_summary.select(len(), sum(total_loss), sum(event_count)) for summary EP stats. collect() materializes loss_summary and summary_ep_stats. If requested, OpenPyXL writes two workbook sheets.In-memory report artifact and optional XLSX file.

6. Important boundaries

  • All model transformations are Polars LazyFrame operations until report collection or collect_loss_summary().
  • Preflight validation runs before any staging, intermediate, mart, or report transformation.
  • Strict schema validation rejects undeclared columns as well as missing or mistyped required columns.
  • No SQL statements are generated by this restored pipeline path.
