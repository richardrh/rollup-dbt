02 / 04 Correctness

Fail fast, not three joins later.

Every column is an enum. Every frame has a declared schema. Every stage validates its inputs and outputs at the boundary. Drift blows up on the line where it happens — with a named diff.

Column names as enum class RefLobsCol(StrEnum): LOB_ID = "lob_id" MODELLED_LOB = "modelled_lob" ROLLUP_LOB = "rollup_lob" LOB_TYPE = "lob_type" CDS_CAT_CLASS_NAME = "cds_cat_class_name" OFFICE = "office" CLASS = "class"
Typed schema per frame REF_LOBS: pl.Schema = pl.Schema({ RefLobsCol.LOB_ID: pl.Int64, RefLobsCol.MODELLED_LOB: pl.String, RefLobsCol.ROLLUP_LOB: pl.String, RefLobsCol.LOB_TYPE: pl.String, RefLobsCol.CDS_CAT_CLASS_NAME: pl.String, RefLobsCol.OFFICE: pl.String, RefLobsCol.CLASS: pl.String, })
01

Typos don't compile. Column names are Python enum members, not strings scattered across SQL files.

02

Schema drift surfaces at the boundary. validate_schema() runs on every stage's inputs and outputs — errors name the frame, the missing columns, and the dtype mismatches.

03

Seeds are schema-checked at load. A renamed column in a CSV stops the pipeline before any math runs.

04

57 unit tests, 1.4 s. The whole contract is exercised on every change, not just in production.

If it's wrong, it fails loudly, at the line where it became wrong.
